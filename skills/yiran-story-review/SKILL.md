# yiran-story-review

对已生成的故事进行质量审核，不通过则自动润色，循环最多 3 次，最后交给人审确认。

## 前置条件

- 项目路径：`/Users/niuniu/Library/Mobile Documents/com~apple~CloudDocs/obsidian-space/30 项目/熠然之智/`
- `source/story.md` 已存在
- `source/prompt.md` 中存在 `series:` 字段标明系列

---

## Step 1：确定系列与审核标准

读取 `source/prompt.md` 中的 `series:` 字段，确定当前故事属于哪个系列，使用对应的审核标准。

### 🔬 深度系列审核标准

| # | 审核项 | 说明 |
|---|-------|------|
| 1 | 概念级別 | 概念是否为研究生/前沿理论水平？ |
| 2 | 隐喻准确性 | 隐喻是否准确对应概念核心，无学术硬伤？ |
| 3 | 叙事结构 | 是否先讲寓言后揭示概念，不提前剧透？ |
| 4 | 概念解释 | 结尾是否有清晰的概念解释和隐喻对照表？ |
| 5 | 字数 | 1500-2500 字？ |
| 6 | 可读性 | 没有学术黑话堆砌，外行能看懂？ |

### 🧠 通识系列审核标准

| # | 审核项 | 说明 |
|---|-------|------|
| 1 | 生活关联 | 成年人能否在故事中找到自己生活的影子？ |
| 2 | 概念通俗化 | 复杂概念是否用生活场景讲清楚了？ |
| 3 | 叙事结构 | 是否先讲寓言后揭示概念？ |
| 4 | 概念解释 | 结尾是否有概念解释和隐喻对照？ |
| 5 | 字数 | 1500-2000 字？ |
| 6 | 无说教感 | 是启发而非教导？ |

### 🌱 成长系列审核标准

| # | 审核项 | 说明 |
|---|-------|------|
| 1 | 年龄层 | 主角是否为中学生（12-15 岁）？场景是否贴近校园/家庭？ |
| 2 | 困境真实性 | 选择的困境是否真实可信？没有标准答案？ |
| 3 | 主角人设 | 主角是否不完美、有犹豫有害怕？ |
| 4 | 说教感 | 是否开放式结尾，没有给出"正确答案"？ |
| 5 | 字数 | 800-1200 字？ |
| 6 | 隐喻对照 | 结尾是否有隐喻对照表？ |

## Step 2：检查状态

读取 `source/story.md` 的 frontmatter，确认 `status: draft`。
如果 status 不是 `draft`，说明已经审核过或已发布，拒绝执行并提示。

---

## Step 3：逐项审核

逐条检查，记录每条的 **通过 ✅** 或 **不通过 ❌**。

**字数统计**：用 Python 单行计算，避免 shell pipeline 兼容性问题：
```
python3 -c "import re; s=open('source/story.md').read(); s=re.sub(r'^---.*?---\s*','',s,count=1,flags=re.DOTALL); s=re.sub(r'[\s#*_>\-—\`\[\]()|]','',s); print(len(s))
```

**不通过的项必须说明具体原因。** 例如：

```
❌ 字数：当前 1800 字，成长系列要求 800-1200 字
❌ 主角人设：主角过于完美，没有任何犹豫和缺陷
```

---

## Step 4：第一次润色前备份

在第一次修改 `source/story.md` 之前，先复制原始版本：

- 复制 `source/story.md` → `source/story.original.md`
- 保留原始 prompt 无需改动

以后每次润色都基于最新版修改，不再重复备份。

---

## Step 5：自动润色 + 再审核（循环）

**如果全部通过 → 跳到 Step 6**

**如果有不通过项：**

1. 针对不通过项进行润色，修改 `source/story.md`
2. 每次润色后**更新 story.md 的 frontmatter**：
   - `status: reviewing`
   - `review_rounds: N`（当前轮次）
3. 重新执行 Step 3 审核
4. **最多循环 3 次**
5. 每次循环记录：`第 N 次润色 → 通过 X 项 / 不通过 X 项`

**3 次循环后仍有不通过项 → 记录剩余问题，跳到 Step 6**

---

## Step 6：提交人审

更新 story.md frontmatter：
- `status: reviewing`（等待人审）

输出审核报告：

```
## 审核报告

系列：🌱 成长系列
概念：选择
轮次：共 X 轮审核（含 Y 次润色）

### 审核结果

| # | 审核项 | 状态 |
|---|-------|------|
| 1 | 年龄层 | ✅ |
| 2 | 困境真实性 | ✅ |
| 3 | 主角人设 | ✅ |
| 4 | 说教感 | ❌ → 已润色 ✅ |
| 5 | 字数 | ✅ |
| 6 | 隐喻对照 | ✅ |

### 润色记录

- 第 1 次润色：调整了主角性格，增加了犹豫描写
- 第 2 次润色：（如适用）

### 待确认

- [ ] 以上审核通过，请用户确认 ✅
- [ ] （如有未通过项）以下问题请用户判断是否接受：
  - XXX（原因说明）
```

**提交给人审后等待用户确认。**

- **用户确认通过** → 更新 story.md frontmatter：
  - `status: reviewed`
  - `review_rounds: N`（最终轮次）
  - 保留其他字段不变
  - 同时自动更新索引文件：
    - `🧠通识系列.md` / `🔬深度系列.md` 等系列页：`草稿 📝` → `已审核 ✅`
    - `_todo.md`：`待审核` → `已审核 ✅`
  - 报告完成
- **用户要求再改** → 按用户要求修改，重新提交人审（不增加 review_rounds 计数）
- **用户要求重写** → 标记 status 为 draft，清空 review_rounds，退回 Step 1（故事生成）
