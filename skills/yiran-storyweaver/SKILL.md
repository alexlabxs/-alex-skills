# yiran-storyweaver

为「熠然之智」公众号生成系列寓言故事。

## 工作流

### Step 0：确定系列和概念

问用户：
1. **系列**：深度 / 通识 / 成长 / 童心 / 社会
2. **概念**：从该系列概念池选一个，或让用户自由输入

### 系列概念池

| 系列 | 概念池 | 叙事方式 | 字数 | 目标受众 |
|------|--------|---------|------|---------|
| **🔬深度** | 叙事谬误、可证伪性、前景理论、博弈论、确认偏误、黑箱问题、对齐问题、因果推理、涌现行为 | 跨学科交叉、方法论反思、学术隐喻 | 1500-2500 | 知识从业者 |
| **🧠通识** | 批判思维、机会成本、路径依赖、激励机制、锚定效应、社会规范 | 系统性思考、跨学科视角、紧贴生活场景 | 1500-2000 | 大众读者 |
| **🌱成长** | 友谊、选择、自我认知、偏见、从众、独立思考、身份认同 | 校园/家庭场景、两难困境、不完美主角 | 800-1200 | 中学生·家长 |
| **🧸童心** | 诚实、分享、勇气、好奇心、情绪认知、轮流、分离焦虑 | 动物角色、简单因果、重复句式、美好结局 | 300-500 | 家长（亲子共读） |
| **🌏社会** | 公地悲剧、群体极化、社会规范、信任机制、信息茧房、旁观者效应 | 公共议题、群体行为、制度反思、宏观叙事 | 1500-2500 | 关心时事的大众 |

### Step 1：生成故事

使用以下 prompt 生成故事：

```
请你从{系列}的概念池里选择{概念}。
然后写一个寓言故事，用间接的方式把这个概念讲清楚。
不要一开始就说答案，尽量到故事快结束的时候，才让人意识到原来讲的是这个概念。

系列要求：
- 叙事方式：{叙事方式}
- 字数：{字数}
- 目标受众：{受众}

故事结束后，再解释这个概念，以及故事里的隐喻分别对应什么。
```

### Step 2：创建内容目录

项目根目录：
```
/Users/niuniu/Library/Mobile Documents/com~apple~CloudDocs/obsidian-space/30 项目/熠然之智/
```

目录命名规则：`YYYYMMDD-序号-[系列]故事标题`

例如：`0615-02-[成长]林小天的两个选择`

> **Slug 标题生成规则**：故事标题取自生成的故事 H1 标题。如果概念不在概念池中或标题太长，取故事标题前 4~8 个字作为 slug。不要用英文或拼音，保持中文。

创建子目录（注意：01-wechat/ 02-xiaohongshu/ 03-video/ 在根目录下，不在 source/ 内）：
```
source/                  ← 仅存放 story.md + prompt.md
01-wechat/assets/        ← 公众号配图
02-xiaohongshu/assets/   ← 小红书素材
03-video/assets/         ← 视频素材
```

### Step 3：保存文件

story.md 必须包含 frontmatter：

```markdown
---
title: 林小天的两个选择
series: 成长
status: draft
review_rounds: 0
created: 2026-06-15
---
```

| 文件 | 内容 |
|------|------|
| `source/prompt.md` | 本次生成使用的 prompt |
| `source/story.md` | 带 frontmatter 的原始故事 |

### Step 4：更新索引（一次性完成）

用单次批量写入完成以下 3 项索引更新（不分步执行）：

1. **对应系列页**（如 `🧠通识系列.md`）→ 添加一行作品列表
2. **`_index.md` 内容日历** → 添加一行日历记录
3. **`_todo.md` 待办** → 添加公众号/小红书/视频 3 个待办项

## 输出交付

完成后向用户报告：
- 系列 + 概念
- 故事标题
- 目录路径
- 下一步建议（公众号改写 / 小红书 / 视频）
